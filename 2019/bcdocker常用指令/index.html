Docker常用指令

Written by TG🍬ZL with ♥ on 17 September 2019 in 爱编程 2 min

容器的基本使用命令

权限管理

1.创建docker组：sudo groupadd docker
2.将当前用户加入docker组：sudo gpasswd -a ${USER} docker
3.重启服务：sudo service docker restart
4.刷新docker成员：newgrp - docker

更换数据源

sudo mkdir -p /etc/docker 
sudo tee /etc/docker/daemon.json <<-'EOF' 
{ 
    "registry-mirrors":["https://rmvesv1w.mirror.aliyuncs.com"] 
}
EOF
sudo systemctl daemon-reload   
sudo systemctl restart docker

常用命令

ps -ef|grep docker 查看运行的进程

service docker restart 重启

docker配置文件的地址 /ect/docker

systemctl daemon-reload
systemctl restart docker 重新加载配置文件和重启

docker version 查看当前的版本信息
docker info 查看下载的docker的详细信息
docker –help 帮助命令

docker images 查看本机的镜像模版

docker images

-repositore： 表示镜像的仓库源
-tag： 镜像的标签，镜像：标签
-iamge ID： 镜像的id
-created ： 镜像的创建时间
-size： 镜像的大小
-a 列出本地所有的镜像（含有中间映像层）
-q 只显示镜像的id（用于批量操作使用）
–digests 显示镜像的摘要信息
–no-trunc 显示镜像的完整信息

docker search 查找某镜像（按照名字） 拉取从dockerhub

-s start的数量 后接超过的数量（-s 30 )超过30 docker search -s 30 Tomcat
–automated 列出automated build类型的镜像

docker pull 拉取镜像文件 docker pull tomact[:标签]（latest最新的）
docker rmi 删除本地的镜像[:latest]

-f force 强制删除
[arg..] 删除多个
-f $(docker images -q)删除全部的镜像

docker run 启动镜像文件

–name=“容器的名称” 起一个别名
-d 后台运行容器，并返回容器的ID，即启动守护线程式的容器
-i 以交互模式启动，通常与-t 同时使用
-t 为容器分配一个伪输入窗口，通常与-i一起使用
-P 随机端口映射
-p：ip：host：container 指定端口映射

ip：：container
host：container
container

docker ps 列出当前正在运行的容器

container id 容器id
image 镜像的编号
command 执行的命令
created 创建的时间
sratus 当前的的状态
ports 端口映射信息
names 容器的名称（–name自己定义）
-a 列出所有正在运行的和历史记录
-l 显示最近创建的容器（-lq只显示编号）
-n 显示最近的n个容器
-q 静默模式，只显示容器的编号（批量处理是使用）
–no-trunc 不截断输出

退出容器 docker run -it 容器

exit 容器停止退出
ctrl+P+Q 容器不停止退出
启动容器 docker start 容器 启动容器/id
重启容器 docker restart 容器 重启容器
停止容器 docker stop 容器 停止容器
强制停止 docker kill 容器 强制停止
删除容器 docker rm 容器 删除容器

（删除全部的）-f强制删除
docker rm -f $(docker ps -a -q)
docker ps -a -q | xargs docker rm

启动式交互

docker run -it 容器 交互式容器
docker run -d 容器 后台运行 ps查看已经退出了（Docker容器后台运行，必须有一个前台进程）

使用shell，让容器执行一个打印的命令：
/bin/sh -c “while true ;do echo hello word ;sleep 2;done”
这样就能看到ps在运行了，使用logs -f实时打印日志信息

查看容器日志

docker logs -f -t –tail 容器id 查看容器的日志信息

-t 加入时间戳（显示时间）
-f 跟随最新的日志打印
–tail 数字显示最多多少行（显示最后的几行）

查看容器运行的程序

docker top 容器id 查看容器运行的进程

查看容器内部的细节

docker inspect 容器id 查看容器内部的细节信息

进入正在运行的容器并以命令交互

docker exec -it 容器id bashshell 交互容器运行shell 不能进入已经停止的，或者已经杀死的(使用-it否则会卡顿)
docker attach 容器id 重新进入 不能进入已经停止的容器

区别：exec是在容器中打开新的终端， attach直接进入容器的
并且可以启动新的进程（可以进入容器 启动命令的终端，不会 中，也可以打印容器执行的结果不进入 启动新的进程。（会进 ） 入容器）

从容器内拷贝文件到主机上

docker cp 容器id：容器目录 主机目录 复制文件到主机上 copy的命令
