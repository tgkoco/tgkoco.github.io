gomod环境的配置和简单使用

环境的配置

  • 要开始使用 go module 的特性， 需要先设置 GO111MODULE 环境变量。

  • 在环境变量中新建GO111MODULE，值为on（GO111MODULE=on）。

  • 开启GOPROXY（原因是无法访问）

  • 环境变量中新建配置GOPROXY ，设置为https://goproxy.io， 或者 https://mirrors.aliyun.com/goproxy/ （推荐使用阿里的）（GOPROXY=https://mirrors.aliyun.com/goproxy/ ）。

  • 环境变量的配置结果

简单使用

第一种方式（使用golang集成开发工具）

  • 选择 创建 go module项目

  • 在代理中输入 https://goproxy.io ，或者 https://mirrors.aliyun.com/goproxy/

  • 就可以创建一个mod项目。

  • 或者在设置里面配置GO Modules

  • 项目目录执行go mod init进行初始化

    第二种方式 （go mod init）

  • 配置完成环境的搭建

  • 创建一个项目,使用 go mod init进行初始化

  • 项目目录执行go get XXXXX，可以导入XXXX包到项目中。

  • 写好代码，执行go run 就可以启动项目。

    提示信息

    go mod 下载的库，都存储在
    $GOPATH/pkg/mod/cache 下
    导入的工程含有mod文件,需要的依赖程序 go mod download或者go run

    下面我们来了解一下go mod的相关命令。

    init 初始化go module download 下载go.mod中的依赖到本地的缓存目录中（$GOPATH/pkg/mod）下 edit 编辑go.mod，通过命令行手动升级和获取依赖 vendor 将项目依赖拷贝到vendor下 tidy 安装缺少的依赖，舍弃无用的依赖 graph 打印模块依赖图 verify 验证依赖是否正确

#### 测试代码

在项目文件的根目录打开命令行窗口：go get github.com/gin-gonic/gin

就可以看到go.mod发生了变化，以及生成了go.sum(可以删除，执行go run之后还会生成)

创建一个main.go文件，输入以下代码并运行。

package main

import "github.com/gin-gonic/gin"

func main() {
    r := gin.Default()
    r.GET("/", func(c *gin.Context) {
        c.String(200, "Hello Gin")
    })
    r.Run() // listen and serve on 0.0.0.0:8080
}
