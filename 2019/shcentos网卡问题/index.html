Centos网卡问题

Written by TG🍬ZL with ♥ on 21 October 2019 in 爱生活 1 min

Centos7网卡

更改网卡名称

1.改名
mv /etc/sysconfig/network-scripts/ifcfg-eno1 /etc/sysconfig/network-scripts/ifcfg-eth0
2.修改grup
vi /etc/default/grub
添加 "net.ifnames=0 biosdevname=0"
即为： GRUB_CMDLINE_LINUX="crashkernel=auto rd.lvm.lv=centos/root rd.lvm.lv=centos/swap rhgb quiet"
----------->
GRUB_CMDLINE_LINUX="crashkernel=auto rd.lvm.lv=centos/root net.ifnames=0 biosdevname=0 rd.lvm.lv=centos/swap rhgb quiet"
3.更新grub配置和内核参数
grub2-mkconfig -o /boot/grub2/grub.cfg
4.重启
reboot

配置静态地址

1.更改配置信息
vi /etc/sysconfig/network-scripts/ifcfg-eth0
更改如下内容：
BOOTPROTO=static
ONBOOT=yes
IPADDR=192.168.5.12
NETMASK=255.255.255.0
GATEWAY=192.168.5.1
2.修改dns
vi /etc/resolv.conf
添加如下（使用的阿里的DNS服务）：
nameserver 223.5.5.5
nameserver 223.6.6.6

配置bond模式

以网卡 eth0、 eth1 绑定命名为 bond1 为例， bond 配置步骤：

1.修改网卡 eth0 vi /etc/sysconfig/network-scripts/ifcfg-eth0
TYPE=Ethernet
BOOTPROTO=none
DEVICE=eth0
ONBOOT=yes
USERCTL=no
MASTER=bond1
SLAVE=yes
2.同上步骤修改 eth1/etc2……
3.新建 bond1 配置 vi /etc/sysconfig/network-scripts/ifcfg-bond1
TYPE=Bond
BOOTPROTO=none
DEVICE=bond1
ONBOOT=yes
IPADDR=192.168.8.8
NETMASK=255.255.255.0
GATEWAY=192.168.8.1
USERCTL=no
BONDING_MASTER=yes
BONDING_OPTS="mode=1 miimon=100"
4.创建 bonding.conf 文件 vi /etc/modprobe.d/bonding.conf（可跳过）
alias bond1 bonding
options bond1 mode=1 miimon=100
5.执行命令 modprobe bonding 更新加载 bonding 模块
执行命令 lsmod | grep bonding 如出现如下结果，说明加载成功
6.重启网络服务
systemctl stop NetworkManager
systemctl disable NetworkManager
systemctl restart network
7.查看绑定模式及网卡状态信息 
cat /proc/net/bonding/bond1
